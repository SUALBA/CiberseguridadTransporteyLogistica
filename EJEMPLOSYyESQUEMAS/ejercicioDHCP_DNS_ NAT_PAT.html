DHCP · Configuración rápida

Red doméstica con laptop, smartphone y consola. El router actúa como servidor DHCP y entrega IPs automáticamente.

ip dhcp pool CASA
 network 192.168.1.0 255.255.255.0
 default-router 192.168.1.1
 dns-server 192.168.1.1
 lease 7

Resultado esperado: Laptop → 192.168.1.2 · Smartphone → 192.168.1.3 · Consola → 192.168.1.4

Ver flujo DORA

Puertos UDP: 67 (servidor), 68 (cliente).

NAT · Traducción privada ↔ pública

Empresa con 3 PCs (192.168.1.2, .3, .4) y una sola IP pública 203.0.113.1.

ip nat inside source list 1 interface FastEthernet0/0 overload
access-list 1 permit 192.168.1.0 0.0.0.255

Tipos: Static 1:1 · Dynamic rango↔rango · Overload (PAT).

Checklist de buenas prácticas

  • Limita el pool DHCP y usa reservas para dispositivos críticos.
  • Activa DHCP Snooping y bloquea puertos no confiables.
  • Implementa DNSSEC y registra/monitoriza consultas sospechosas.
  • Usa NGFW con inspección y IDS/IPS para NAT/PAT.
  • Segmenta por VLAN: usuarios / servidores / IoT / gestión.
  • Protege salidas con ACLs y políticas de mínimo privilegio.
  • Para sedes remotas: VPN + (opcional) GRE si necesitas routing dinámico.

Notas rápidas para examen

  • DORA = Discover · Offer · Request · Acknowledge.
  • DNS usa UDP/53 y TCP/53 (zonas).
  • NAT opera en Capa 3 (traduce direcciones IP).
  • PAT mapea IP:PUERTO ↔ IP interna:PUERTO.
  • NAT ≠ seguridad: añade firewall, IDS/IPS.

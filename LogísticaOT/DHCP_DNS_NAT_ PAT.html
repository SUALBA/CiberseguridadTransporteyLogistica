DHCP, DNS, NAT y PAT en Redes Modernas

Introducción

En el ámbito de la ciberseguridad, el conocimiento de protocolos y servicios como DHCP, DNS, NAT y PAT es esencial. Estos mecanismos constituyen la base de las comunicaciones modernas en redes domésticas y empresariales. Sin embargo, no solo garantizan la conectividad: una configuración inadecuada puede abrir vulnerabilidades críticas explotables por atacantes.

Objetivo: analizar cómo funcionan DHCP, DNS, NAT y PAT, mostrar configuraciones reales en routers Cisco y destacar sus implicaciones en ciberseguridad.

1. DHCP (Dynamic Host Configuration Protocol)

Concepto

DHCP automatiza la asignación de direcciones IP, máscaras, puertas de enlace y DNS. Facilita la administración en redes grandes, evitando configuraciones manuales.

Funcionamiento

  1. Discover: el cliente busca un servidor DHCP.
  2. Offer: el servidor ofrece una dirección IP.
  3. Request: el cliente solicita la IP ofrecida.
  4. Ack: el servidor confirma la asignación.

Ejemplo práctico

En una red doméstica, un router asigna:

  • Laptop → 192.168.1.101
  • Smartphone → 192.168.1.102
  • Consola → 192.168.1.103
ip dhcp pool RED_LOCAL
network 192.168.1.0 255.255.255.0
default-router 192.168.1.1
dns-server 8.8.8.8
lease 7

Riesgos de seguridad

  • DHCP Spoofing: un atacante actúa como servidor falso.
  • IP Starvation: ataques que agotan las direcciones IP disponibles.

2. DNS (Domain Name System)

Concepto

DNS traduce nombres de dominio en direcciones IP. En redes internas también permite resolver nombres locales como miportal → 192.168.1.5.

Configuración en Cisco

ip dns server
ip host miportal 192.168.1.5

Riesgos

  • DNS Spoofing: respuestas falsas que redirigen tráfico.
  • DNS Tunneling: uso de DNS para exfiltrar datos.

3. NAT (Network Address Translation)

Concepto

NAT permite que múltiples equipos con IP privadas accedan a Internet usando una sola IP pública.

Escenario

PC1 (192.168.1.2) ─┐
PC2 (192.168.1.3) ─┼──> Router → Internet
PC3 (192.168.1.4) ─┘ |
└─> 203.0.113.1
interface GigabitEthernet0/0
ip address 192.168.1.1 255.255.255.0
ip nat inside
!
interface GigabitEthernet0/1
ip address 203.0.113.1 255.255.255.0
ip nat outside

access-list 1 permit 192.168.1.0 0.0.0.255
ip nat inside source list 1 interface GigabitEthernet0/1 overload

4. PAT (Port Address Translation)

Concepto

PAT (NAT con sobrecarga) permite que varios dispositivos compartan la misma IP pública diferenciándose por los puertos.

Tabla de traducciones

PC2 192.168.1.3:5002 203.0.113.1:30002

show ip nat translations

Conclusiones

- DHCP facilita la gestión de IPs, pero debe protegerse contra spoofing.
- DNS es esencial para la comunicación, pero es una vía común de ataques.
- NAT optimiza el uso de direcciones IPv4, aunque no es una medida de seguridad en sí misma.
- PAT permite compartir una IP pública entre múltiples equipos, pero añade complejidad al análisis de tráfico.

En conjunto, estos mecanismos son piezas fundamentales de la arquitectura de red y deben entenderse tanto desde su función operativa como desde su impacto en la seguridad.
